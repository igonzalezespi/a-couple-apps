# Security Policy

## Supported Versions

Only the latest `develop` and `main` branches are supported with security patches.

| Version | Status      |
|---------|-------------|
| main    | Supported   |
| develop | Supported   |
| older   | Unsupported |

## Reporting a Vulnerability

Please report security vulnerabilities privately to protect users. **Do not** open a public issue.

Use GitHub's private vulnerability reporting:

1. Navigate to the repository's **Security** tab.
2. Select **Report a vulnerability**.
3. Provide details: description, affected versions, and impact assessment.

Public issues disclosing vulnerabilities without a patch will be closed.

## Disclosure and Coordination

- Maintainers acknowledge reports within a reasonable timeframe.
- Patches are developed, tested, and released before public disclosure.
- We follow coordinated disclosure practices to give users time to update.
